# Issue Labels and Milestones

Phase 0 proposal for GitHub labels and milestones. This document is a template; labels
must be created in the GitHub UI or via automation.

## Milestone labels

- `milestone:M0` - Parity baselines
- `milestone:M1` - Core semantics and UX
- `milestone:M2` - Registry and ecosystem
- `milestone:M3` - Provisioning engine
- `milestone:M4` - Provider coverage
- `milestone:M5` - Security and release

## Area labels

- `area:ansible`
- `area:terraform`
- `area:registry`
- `area:provider-sdk`
- `area:cli`
- `area:security`
- `area:docs`

## Status labels

- `status:blocked`
- `status:needs-spec`
- `status:ready`
- `status:in-progress`
- `status:needs-review`

## How to apply

1. Create milestones M0 through M5 in the GitHub UI.
2. Create the labels above (or your own equivalents).
3. Tag existing gaps and roadmap items with the appropriate milestone and area label.
